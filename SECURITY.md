# Security Policy

## Supported Versions

VRC-Sentry is currently in active development (**Alpha**). Security patches are prioritized for the latest pre-release version.

| Version | Supported |
| ------- | ------------------ |
| < 1.0.0 | :white_check_mark: |
| > 1.0.0 | :x:                |

## Reporting a Vulnerability

I take the security of my projects seriously. If you **believe** you have found a security vulnerability, please help me by reporting it responsibly.

### How to report

Please **do not** open a public issue for security vulnerabilities. Instead, use one of the following private channels:

* **E-Mail**: [report@lunared.work](mailto:report@lunared.work)
* **GitHub Security Advisory**: If available on this repository, please use the "Report a vulnerability" button under the **Security** tab.

**Please include the following in your report:**

1. A brief description of the **vulnerability**.
2. Steps to reproduce (or a proof-of-concept).
3. The potential impact of the exploit.

---

## What to expect

Once a report is received, I will follow this process:

### 1. Acknowledgement

You will receive an initial response from me within **24-72 hours** **acknowledging** that I have received your report.

### 2. Evaluation

I will investigate the report to confirm the vulnerability. During this time, I may contact you for further clarification. I aim to complete the evaluation within **7 business days**.

### 3. Resolution & Updates

* **Accepted:** If the vulnerability is confirmed, I will work on a fix. You will receive status updates at least once a week until the issue is resolved.
* **Declined:** If I determine it is not a vulnerability (e.g., intended **behaviour or** out of scope), I will provide a detailed explanation.
* **Public Disclosure:** Once a fix is deployed, I will coordinate with you to decide on a public disclosure date. I prefer **coordinated disclosure** to ensure users have time to update.
